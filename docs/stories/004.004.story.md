# User Story: CALE-004

**Epic:** EPIC-004 — Scheduler & Appointment Management
**Title:** Appointment Status Management — Confirm, Reschedule, Cancel
**ID:** CALE-004
**Created:** 2026-05-15
**Status:** Draft
**Phase:** 2

---

## User Story Statement

**As a** clinic staff
**I want** to change appointment status (scheduled, confirmed, completed, cancelled)
**So that** I can track appointment lifecycle and patient communication

---

## Acceptance Criteria

- [x] Status options: Scheduled, Confirmed, Completed, Cancelled, No-show, Rescheduled
- [x] Status actions available from calendar/appointment details
- [x] Reschedule: Opens date/time picker, creates new slot, marks old as "rescheduled"
- [x] Cancel: Requires reason (dropdown), confirmation dialog
- [x] Completed: Mark done with optional notes
- [x] No-show: Track patient absence
- [x] Status change log: Shows who changed what when (mock data)
- [x] No API calls (Phase 2)
- [x] Notifications: Show status change UI updates (no email/SMS Phase 2)

---

## Technical Requirements

**Frontend:**
- Create `/components/scheduler/AppointmentActions.tsx`
- Status state: Enum (SCHEDULED, CONFIRMED, COMPLETED, CANCELLED, NOSHOW, RESCHEDULED)
- Action buttons: Confirm, Reschedule, Cancel, Mark Done, Mark No-show
- Confirmation dialogs: React-based (no window.confirm)
- Reschedule: Reuse AppointmentForm component
- Cancel reason: Dropdown (No-show, Doctor unavailable, Patient request, Emergency, Other)
- Change log: Display in appointment details modal

**Styling:**
- Action buttons: Color-coded by status (green=confirm, red=cancel)
- Status badge: Visually distinct for each status
- Modal for confirmations

**Testing:**
- Unit: Status transitions valid
- Integration: Status change reflected in calendar
- Visual: Action buttons responsive
