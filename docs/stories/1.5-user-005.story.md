# User Story: USER-005

**Epic:** EPIC-001 — Authentication & User Management
**Title:** Permission Assignment & Clinic Admin Dashboard
**ID:** USER-005
**Created:** 2026-05-14
**Status:** Ready for Review

## Overview

Implement admin dashboard for clinic administrators to view, assign, and manage user roles and permissions. Includes audit log of role changes for compliance.

---

## User Story Statement

**As a** clinic admin
**I want** to assign roles to team members and manage their permissions
**So that** I can control who has access to what features in the clinic system

---

## Acceptance Criteria

- [x] Admin dashboard page `/app/admin/users` lists all clinic users with roles
- [x] Admin can assign roles to users (Doctor, Receptionist, Patient)
- [x] Admin can remove roles from users
- [x] Role changes trigger audit log entry (user, change, timestamp)
- [x] Audit log visible to admins only at `/app/admin/audit-log`
- [x] Cannot remove admin role if only one admin exists (safeguard)
- [ ] Bulk role assignment for multiple users (future epic candidate)

---

## Technical Requirements

**Frontend:**
- Create `/app/admin/users` page with user table:
  - Columns: Name, Email, Clinic, Current Role(s), Actions
  - Search/filter by name or email
  - Pagination (20 users per page)
- Role assignment modal/form
- Audit log view at `/app/admin/audit-log`
- Bulk action checkboxes (optional for MVP)

**Backend:**
- Admin-only middleware: Check if user has admin role
- Endpoint: PUT `/api/admin/users/{user_id}/roles` to assign role
- Endpoint: DELETE `/api/admin/users/{user_id}/roles/{role_id}` to remove role
- Audit logging endpoint: GET `/api/admin/audit-log`
- Validation: Only admins can modify roles, cannot unassign last admin

**Database:**
- Create `audit_log` table:
  ```sql
  CREATE TABLE public.audit_log (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    actor_id UUID NOT NULL REFERENCES users(id),
    action TEXT NOT NULL, -- "role_assigned", "role_removed"
    target_user_id UUID NOT NULL REFERENCES users(id),
    target_role_id UUID REFERENCES roles(id),
    clinic_id UUID NOT NULL,
    created_at TIMESTAMP DEFAULT now()
  );
  ```
- RLS policy: Only admins of the clinic can view their clinic's audit log

**Security:**
- Admin-only access: Middleware checks user role before rendering page
- SQL injection prevention: Parameterized queries
- Privilege escalation prevention: Users cannot modify their own role
- Audit trail: All role changes logged with actor, target, timestamp

**Testing:**
- Unit test: Role assignment logic, audit log creation
- Integration test: Assign role → verify in database and audit log
- Security test: Non-admin access attempt (blocked), privilege escalation
