# User Story: CALE-007

**Epic:** EPIC-004 — Scheduler & Appointment Management
**Title:** Appointment History & Analytics
**ID:** CALE-007
**Created:** 2026-05-15
**Status:** Draft
**Phase:** 2

---

## User Story Statement

**As a** clinic manager
**I want** to view appointment history and analytics (completion rates, no-shows, duration)
**So that** I can understand scheduling patterns and identify improvements

---

## Acceptance Criteria

- [ ] History page at `/app/scheduler/history` showing past appointments
- [ ] Filter: Date range, doctor, patient, status
- [ ] Search: Patient name, appointment ID
- [ ] View appointment details: Full history of status changes
- [ ] Analytics dashboard: No-show rate, average duration, busiest days, completion rate
- [ ] Export option: Download history as CSV (Phase 2 PDF)
- [ ] No API calls (Phase 2)
- [ ] Mock data: 100+ past appointments

---

## Technical Requirements

**Frontend:**
- Create `/app/scheduler/history/page.tsx`
- Create `/components/scheduler/HistoryTable.tsx` (sortable, filterable)
- Create `/components/scheduler/HistoryAnalytics.tsx` (stats cards + charts)
- Filters: DatePicker (from/to), Doctor dropdown, Status multiselect
- Search: Input field (debounced, searches name and ID)
- Sort: Clicking column headers (date, doctor, patient, status, duration)
- Analytics:
  - No-show rate: Count (NOSHOW) / total %
  - Completion rate: Count (COMPLETED) / total %
  - Avg duration: Mean appointment duration
  - Busiest doctor: Appointment count per doctor
  - Busiest day: Appointments per day of week
- Export: CSV download with all columns (Phase 2: PDF)

**Styling:**
- History table: Responsive on mobile (horizontal scroll)
- Analytics: Card layout with stats + mini charts (Recharts)
- Filters: Sticky top bar

**Testing:**
- Unit: Filter logic, analytics calculation
- Integration: History page loads, filters work
- Visual: Responsive on mobile/desktop
