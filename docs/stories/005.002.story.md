## User Story Statement

**As a** security officer
**I want** payment method data to be tokenized and stored securely
**So that** we remain PCI DSS compliant

---

## Acceptance Criteria

- [ ] Stripe Payment Method tokens stored (no card data in DB)
- [ ] Saved payment methods listed in patient profile
- [ ] Delete saved payment method endpoint working
- [ ] Payment method used as default for future transactions
- [ ] Audit log captures all tokenization events
- [ ] No card data ever touches our servers
- [ ] PCI compliance certified by external auditor (if required)

---

## Technical Requirements

**Backend:**
- Use Stripe SetupIntent for payment method creation
- Store `payment_method_id` in patients table
- Endpoint: `POST /api/patient/{id}/payment-methods` (save)
- Endpoint: `DELETE /api/patient/{id}/payment-methods/{methodId}`
- Audit logging in audit_log table

**Security:**
- Never store card numbers, expiry, CVV
- Use Stripe's vaulting only
- Tokenization mandatory before saving

**Testing:**
- Unit: Token creation, deletion
- Integration: Save/list/delete flow
- Security: Verify no card data in logs
