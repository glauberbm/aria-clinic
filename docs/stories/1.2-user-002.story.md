# User Story: USER-002

**Epic:** EPIC-001 — Authentication & User Management
**Title:** Login & Logout Flows with JWT Token Management
**ID:** USER-002
**Created:** 2026-05-14
**Status:** Done

---

## Overview

Implement complete login and logout flows with JWT token management, session persistence, token refresh, and secure logout across all devices.

---

## User Story Statement

**As a** clinic staff member
**I want** to securely log in with my credentials and maintain my session
**So that** I can access the clinic management system without re-entering credentials

---

## Acceptance Criteria

- [x] Login page with email/password fields and "Forgot Password" link
- [x] JWT token received on successful login, stored securely (httpOnly cookie)
- [x] Session persists across page reloads (token refresh)
- [x] Token auto-refresh 5 minutes before expiry (silent refresh)
- [x] Logout clears session and redirects to login
- [x] Failed login shows specific error (wrong password vs. user not found)
- [x] Rate limiting on failed login attempts (5 attempts → 15 min lockout)

---

## Technical Requirements

**Frontend:**
- Create `/app/auth/login` page with form (email, password)
- "Forgot Password" link to password reset workflow
- Persistent session check on app load (redirect if not authenticated)
- Use `@supabase/supabase-js` client for auth operations
- Display "Logging in..." state during authentication

**Backend:**
- Supabase Auth session management (auto-handles JWT)
- Middleware to protect routes (redirect unauthenticated users to /auth/login)
- Token refresh endpoint (optional if using Supabase client-side refresh)
- Logout endpoint to clear server-side session state

**Security:**
- JWT tokens in httpOnly, secure cookies (not localStorage)
- CSRF protection on logout
- Rate limiting: 5 failed attempts → 15-minute IP lockout
- Secure logout: clear all tokens, invalidate server-side session
- No sensitive data in JWT payload (use secure HTTP-only cookies)

**Testing:**
- Unit test: Form validation, error messages
- Integration test: Login → token storage, logout → token cleared
- Security test: Session hijacking attempts, token expiry handling
