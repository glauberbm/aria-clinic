## Title
Integrate ArIA Agent WhatsApp notifications for patient communications

## Description
Connect patient communication system to ArIA Agent WhatsApp integration for automated appointment reminders, treatment updates, and patient engagement.

## Acceptance Criteria

- [x] WhatsApp message templates configured
- [x] Appointment reminders sent 24h before
- [x] Patient opt-in/opt-out preferences respected
- [x] Message delivery tracking
- [x] Conversation history in system
- [x] Error handling and retries

## Implementation Summary

### AC #1: WhatsApp Message Templates ✅
- Created `lib/whatsapp/templates.ts` with 8 pre-configured message templates
- Templates include: appointment reminders (24h, 1h), status updates, treatment updates, follow-ups, medication reminders
- Variables substitution system with validation
- Portuguese (pt_BR) language support for Brazilian clinic context

### AC #2: Appointment Reminders 24h Before ✅
- Created `app/api/patient/whatsapp/send-pending-reminders/route.ts` scheduler endpoint
- Runs hourly to find appointments needing 24h and 1h reminders
- Includes rate limiting and CRON_SECRET authentication
- Tracks reminder status in appointments table

### AC #3: Patient Opt-in/Opt-out Preferences ✅
- Created `components/patient/WhatsAppPreferences.tsx` component
- UI for managing WhatsApp, appointment reminders, marketing, newsletter consents
- Time preference selection (morning, afternoon, evening)
- Database persistence via patient_contact_preferences table

### AC #4: Message Delivery Tracking ✅
- Messages logged to patient_communications table with status tracking
- Status states: pending → sent → delivered → read OR failed
- Timestamps for sent_at, delivered_at, read_at
- Error message tracking for failed deliveries
- Retry count tracking (up to 3 retries)

### AC #5: Conversation History ✅
- Created `components/patient/WhatsAppConversationHistory.tsx` component
- Displays all WhatsApp messages for a patient
- Shows message type, status, delivery timestamps
- Time-ago relative dates (humanized)
- Error messages visible for troubleshooting

### AC #6: Error Handling and Retries ✅
- WhatsApp service implements exponential retry logic (3 attempts, 5sec intervals)
- Graceful handling of missing preferences (creates on first use)
- Non-blocking error logging (failures don't prevent operations)
- User-friendly error messages in UI
- Try-catch blocks on all async operations

## Files Created

**Core WhatsApp Infrastructure:**
- `lib/whatsapp/templates.ts` - Message templates and substitution
- `lib/whatsapp/service.ts` - WhatsApp service class with Supabase integration
- `supabase/migrations/20260520000002_create_appointments_table.sql` - Appointments table with RLS

**API Routes:**
- `app/api/patient/whatsapp/send-reminder/route.ts` - Send individual reminders
- `app/api/patient/whatsapp/send-pending-reminders/route.ts` - Scheduler for batch reminders

**UI Components:**
- `components/patient/WhatsAppPreferences.tsx` - Patient preferences management
- `components/patient/WhatsAppConversationHistory.tsx` - Message history view

## Integration Points

1. **Patient Registration:** Preferences created with default opt-in for appointment reminders
2. **Appointment Creation:** Tracks reminder_sent_24h and reminder_sent_1h flags
3. **Patient Detail Page:** Can embed WhatsAppPreferences and WhatsAppConversationHistory
4. **Admin Dashboard:** Can use send-pending-reminders endpoint in cron job or manual trigger
5. **Patient Settings:** Can manage preferences in account settings

## Environment Configuration Required

- `CRON_SECRET` - Authentication token for scheduler endpoint
- `NEXT_PUBLIC_APP_URL` - App URL for scheduler to call send-reminder endpoint
- `SUPABASE_SERVICE_ROLE_KEY` - For backend operations in WhatsApp service
