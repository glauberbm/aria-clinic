## Overview

Implement user profile page allowing users to view their profile information, edit their details, upload/change avatars, and update clinic association.

---

## User Story Statement

**As a** clinic staff member
**I want** to view and update my profile information
**So that** my personal data is accurate and I can upload my profile picture

---

## Acceptance Criteria

- [ ] Profile page `/app/profile` shows logged-in user's details (name, email, clinic, role)
- [ ] Edit profile form allows updating name and other non-sensitive fields
- [ ] Avatar upload with image preview and crop functionality
- [ ] Save changes persist to public.users table
- [ ] Profile shows user's clinic and assigned role(s)
- [ ] Clinic association displayed but not editable by user (admin only)

---

## Technical Requirements

**Frontend:**
- Create `/app/profile/page.tsx` with user information display
- Create `/app/profile/edit` page with editable form
- Avatar upload with preview (use Uppy or similar)
- Form validation: name length, email format
- Loading states and error messages
- Breadcrumb navigation: Settings > Profile

**Backend:**
- Avatar storage: Supabase Storage (public bucket, user-specific folder)
- Update endpoint: PATCH `/api/profile` with auth check
- RLS policy: Users can only read/update their own profile
- Profile picture URL generation with CDN optimization

**Database:**
- Add `avatar_url` column to users table (nullable, TEXT)
- Add `updated_at` trigger to auto-update timestamp

**Security:**
- File size limit: 5MB for avatars
- File type validation: JPEG, PNG only
- Auth required: Only authenticated users can access profile
- CORS restriction: Avatar URL accessible only to authorized users

**Testing:**
- Unit test: Form validation, avatar handling
- Integration test: Update profile → verify in database
- Security test: Unauthorized profile access, file upload abuse
