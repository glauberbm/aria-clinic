## Overview

Implement appointment booking system allowing patients to schedule appointments with doctors, and clinic staff to manage appointments and send reminders.

---

## User Story Statement

**As a** patient
**I want** to schedule and manage appointments with doctors
**So that** I can receive timely healthcare services

---

## Acceptance Criteria

- [ ] Patient can view available appointment slots
- [ ] Patient can book appointment with preferred doctor
- [ ] Appointment confirmation email sent to patient
- [ ] Clinic staff can view appointment calendar
- [ ] Staff can manage appointments (create, update, cancel)
- [ ] Automatic appointment reminders (24h before)
- [ ] Appointment status tracking (pending, confirmed, completed, cancelled)
- [ ] Reschedule appointment functionality
- [ ] Timezone support for patient scheduling
- [ ] Appointment notes for doctors

---

## Technical Requirements

**Frontend:**
- Create `/app/appointments` page for patients
- Calendar view for available slots
- Appointment booking form
- Appointment history and status tracking
- Staff appointment management dashboard

**Backend:**
- Endpoint: GET `/api/appointments/available` (list available slots)
- Endpoint: POST `/api/appointments` (book appointment)
- Endpoint: PUT `/api/appointments/{appointmentId}` (update)
- Endpoint: DELETE `/api/appointments/{appointmentId}` (cancel)
- Endpoint: GET `/api/appointments` (list user's appointments)
- Email notifications for confirmations/reminders

**Database:**
- Create `appointments` table
- Create `appointment_slots` table (doctor availability)
- Implement RLS for patient access control
