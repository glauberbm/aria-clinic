# User Story: Build Invoice Generation and Delivery System

**Epic:** EPIC-005 — Stripe Payment Integration

**Created:** 2026-05-17
**Status:** Draft
**Ready for:** @po validation

## Overview

Implement automatic invoice generation and delivery system triggered on successful payment. Generates professional PDF invoices with clinic branding, caches in Supabase Storage, and delivers via email with sequential unique invoice numbering and error recovery for orphaned invoices.

---

## User Story Statement

**As a** patient
**I want** to receive a professional PDF invoice automatically after payment
**So that** I have a record of my payment for accounting purposes

---

## Acceptance Criteria

**Invoice Generation:**
- [ ] Invoice generated automatically on appointment completion
- [ ] Invoice PDF created using html2pdf (P95 latency < 5s)
- [ ] PDF includes: clinic name/CNPJ, patient name/CPF, appointment details, amount, payment method
- [ ] Invoice number generated sequential (YYYY-MM-00001 format)
- [ ] Invoice numbers unique and never duplicated (race condition safe)
- [ ] PDF cached in Supabase Storage with immutable URL
- [ ] Invoice record created in invoices table

**Email Delivery:**
- [ ] Email sent to patient automatically
- [ ] Email includes invoice download link (not attachment)
- [ ] Email sent via Resend API
- [ ] Resend endpoint `/api/invoices/resend` for manual retry
- [ ] Email delivery performance P95 < 3 seconds
- [ ] Email status tracked: pending, sent, failed, bounced

**Error Handling:**
- [ ] Orphaned invoice: Payment succeeded but PDF failed
- [ ] Retry logic: Up to 3 retries (exponential backoff 1s, 5s, 30s)
- [ ] If PDF fails after 3 retries: Create invoice with pdf_url = NULL
- [ ] Email sent with dashboard link instead of PDF
- [ ] Clinic staff notified of failure
- [ ] Manual retry button in dashboard

**Logo Management:**
- [ ] Logo uploaded to Supabase Storage
- [ ] Logo constraints: PNG/JPG only, max 500KB, max 1024x1024px
- [ ] Logo embedded as base64 in PDF
- [ ] Validation on upload (format, size, dimensions)

**Database:**
- [ ] invoices table: id, clinic_id, appointment_id, patient_id, payment_id, invoice_number, amount_cents, currency, pdf_url, email_status, created_at, sent_at
- [ ] Indexes on: appointment_id, patient_id, invoice_number (UNIQUE)

**Brazil Requirements:**
- [ ] CPF validation on invoice generation
- [ ] CNPJ validation on invoice generation
- [ ] Portuguese text rendering verified
- [ ] Currency: BRL (Brazilian Real)

**Testing:**
- [ ] Unit tests for PDF generation (≥70% coverage)
- [ ] Integration tests for email delivery
- [ ] Orphaned invoice scenario tested
- [ ] Concurrent invoice number generation (race condition)
- [ ] Logo validation tests (format, size, dimensions)

## Test Scenarios

**3.1: Happy Path — Invoice Generated**
- Payment webhook received
- PDF generated (< 5s)
- PDF cached in Storage
- Invoice record created
- Email sent to patient
- Audit log: invoice_generated, success: true

**3.2: Orphaned Invoice**
- Payment succeeds
- PDF generation fails 3x
- Invoice created with pdf_url = NULL
- Email sent with dashboard link
- Clinic staff notified
- Manual retry available

**3.3: Email Delivery Failure**
- PDF generated successfully
- Email delivery fails
- Retry queue activated
- Resend endpoint triggers retry
- Email eventually sent

**3.4: Concurrent Invoice Numbers**
- 100 payments complete simultaneously
- 100 invoices generated
- All invoice_numbers unique
- No race condition collisions

**3.5: Logo Validation**
- Valid PNG/JPG uploaded: Accepted
- Invalid format (GIF): Rejected (invalid_logo_format)
- Size > 500KB: Rejected (logo_file_too_large)
- Dimensions > 1024x1024: Rejected (logo_dimensions_exceeded)

**3.6: CPF/CNPJ Validation**
- Patient CPF valid: Invoice generated
- Patient CPF invalid: Error (invalid_cpf)
- Clinic CNPJ valid: Invoice generated
- Clinic CNPJ invalid: Error (invalid_cnpj)

**3.7: Invoice Email Resend**
- Invoice already sent
- Manual resend via endpoint
- Email resent from cache
- No PDF regeneration

**3.8: Portuguese Text Rendering**
- Invoice includes Portuguese characters (Á, É, Í, Ó, Ú, Ç)
- PDF renders correctly
- No encoding errors
