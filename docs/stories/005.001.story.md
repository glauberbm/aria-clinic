## User Story Statement

**As a** clinic manager
**I want** to process patient payments via Stripe
**So that** we can collect appointment fees and generate invoices

---

## Acceptance Criteria

- [ ] Stripe account created and API keys configured
- [ ] Payment endpoint created: `POST /api/payments/process`
- [ ] Card tokenization via Stripe.js (no card data stored locally)
- [ ] Payment status saved to appointments table
- [ ] Webhook `/api/webhooks/stripe` receives payment.success events
- [ ] Error handling for declined cards, timeouts, duplicates
- [ ] PCI DSS compliance audit passed
- [ ] Tests cover happy path + error scenarios
- [ ] Success response includes payment receipt ID

---

## Technical Requirements

**Backend:**
- Create Stripe webhook handler in `/api/webhooks/stripe.ts`
- Payment processing: `/api/payments/process` (validates amount, idempotency)
- Store Stripe payment_intent_id in appointments.payment_intent
- Use Supabase Edge Functions or Next.js API routes
- Implement retry logic for webhook delivery failures

**Security:**
- Environment variables: `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`
- Webhook signature validation (Stripe provided)
- Never log full card numbers
- PCI DSS audit required

**Testing:**
- Unit: Payment creation, webhook validation
- Integration: E2E payment flow with Stripe test cards
- Security: PCI audit checklist
