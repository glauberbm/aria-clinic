## Title
Design patient database schema with RLS policies and LGPD compliance

## Description
Create comprehensive database schema for patient management system in Supabase PostgreSQL, including tables for patient records, medical history, treatments, communications, and audit logs. Implement Row-Level Security (RLS) policies to ensure data privacy and LGPD compliance.

## Acceptance Criteria

- [ ] Patient table created with all required fields (name, email, phone, address, date of birth, etc.)
- [ ] Medical history table with treatment records and dates
- [ ] Communications table for appointment notifications and messages
- [ ] Medications/allergies table linked to patient records
- [ ] Contact preferences table for communication opt-ins
- [ ] Audit log table for compliance tracking
- [ ] RLS policies implemented to restrict patient data access by role
- [ ] Indexes created for search performance
- [ ] Migrations documented and versioned
- [ ] Schema reviewed and approved by security team

## Tasks

- [ ] Define patient table structure
- [ ] Define medical history and treatment tables
- [ ] Define communication and preferences tables
- [ ] Implement RLS policies
- [ ] Create database migrations
- [ ] Add sample data for testing
- [ ] Document schema and ERD

## Technical Notes

### Tables Required
1. `patients` - Core patient data
2. `patient_medical_history` - Treatment records
3. `patient_communications` - Messages and notifications
4. `patient_medications` - Medications and allergies
5. `patient_contact_preferences` - Opt-ins for communication channels
6. `patient_audit_logs` - Compliance and access logs

### RLS Policies
- Patients can only see their own records
- Healthcare staff can see patients assigned to them
- Admins can see all records with audit trails
- All access logged for LGPD compliance

### LGPD Compliance
- Data encryption at rest
- Access logging mandatory
- Right to be forgotten (soft delete with archival)
- Data retention policies (configurable)
