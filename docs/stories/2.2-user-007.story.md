# User Story: USER-007

**Epic:** EPIC-002 — Patient Management & Health Records

**ID:** USER-007
**Created:** 2026-05-15
**Status:** Draft

---

## Overview

Enable clinic staff (doctors, receptionists) to create, update, and manage patient medical records including diagnosis, treatment notes, test results, and prescriptions.

---

## User Story Statement

**As a** clinic staff member
**I want** to manage patient medical records and treatment history
**So that** I can provide better patient care and maintain accurate health documentation

---

## Acceptance Criteria

- [ ] Doctor can create/edit patient medical records (diagnosis, treatment notes)
- [ ] Medical records linked to patient profile
- [ ] Test results upload and storage
- [ ] Prescription history tracking
- [ ] Treatment notes with timestamps
- [ ] Only authorized staff can view/edit records
- [ ] Audit trail for all record modifications
- [ ] Patient can view own medical records (read-only)
- [ ] Records searchable by date and type

---

## Technical Requirements

**Frontend:**
- Create `/app/records/{patientId}` page for staff
- Medical records table with filters
- Form to create/edit records
- Patient view of own records (read-only)

**Backend:**
- Endpoint: POST `/api/records` (create record)
- Endpoint: PUT `/api/records/{recordId}` (update record)
- Endpoint: GET `/api/patient/{patientId}/records` (list records)
- Endpoint: GET `/api/records/{recordId}` (view record details)
- RLS: Staff can access all clinic records, patients see own only

**Database:**
- Create `medical_records` table
- Create `test_results` table
- Create `treatment_notes` table
- Implement RLS for role-based access
