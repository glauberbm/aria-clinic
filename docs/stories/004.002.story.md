# User Story: CALE-002

**Epic:** EPIC-004 — Scheduler & Appointment Management

**ID:** CALE-002
**Created:** 2026-05-15
**Status:** Draft
**Phase:** 2

---

## User Story Statement

**As a** clinic staff
**I want** to create and edit appointments with patient details, doctor, and time
**So that** I can schedule patient visits and update appointment info

---

## Acceptance Criteria

- [ ] Appointment form at `/app/scheduler/appointment/new` (create) and `/[id]/edit` (edit)
- [ ] Fields: Patient ID, Doctor, Date, Time, Duration, Notes, Appointment Type
- [ ] Validation: Required fields, date not in past, no doctor double-booking
- [ ] Doctor dropdown: Populated from mock data (name, specialty)
- [ ] DateTime picker: Calendar + time selector with 30-min intervals
- [ ] Submit: Creates/updates appointment, redirects to calendar
- [ ] Error handling: Show validation errors inline
- [ ] No API calls (Phase 2 backend integration)
- [ ] Form state: Persisted during session (not stored)

---

## Technical Requirements

**Frontend:**
- Create `/app/scheduler/appointment/new/page.tsx`
- Create `/app/scheduler/appointment/[id]/edit/page.tsx`
- Create `/components/scheduler/AppointmentForm.tsx`
- Form library: React Hook Form + Zod validation
- DateTime picker: `react-day-picker` + custom time selector
- Validation rules:
  - All fields required
  - Date must be ≥ today
  - Duration: 15min, 30min, 60min options
  - Doctor not double-booked (check against mock data)
- Mock data: 5 doctors with specialties

**Styling:**
- TailwindCSS form styling
- Error messages: Inline (below field) in red
- Submit button disabled while validation fails
- Loading state on submit (simulated 500ms)

**Testing:**
- Unit: Form validation (required, date, doctor conflict)
- Integration: Create appointment flow
- Visual: Form layout on mobile/desktop
