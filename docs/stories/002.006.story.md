# User Story: DASH-006

**Epic:** EPIC-002 — Dashboard

**Status:** Draft
**Ready for:** @po validation

---

## Overview

Refine and optimize the entire dashboard for mobile and tablet devices. Implement touch-friendly interactions, ensure proper spacing and typography on small screens, and verify all components render correctly across breakpoints (375px → 1920px).

---

## User Story Statement

**As a** clinic staff on mobile
**I want** the dashboard to be fully functional and visually polished on my phone
**So that** I can check clinic metrics and patient info while on the go

---

## Acceptance Criteria

- [ ] Mobile layout verified at 375px, 425px breakpoints
- [ ] Tablet layout verified at 768px, 1024px breakpoints
- [ ] Desktop layout verified at 1440px, 1920px breakpoints
- [ ] Touch targets (buttons, links): Minimum 44x44px on mobile
- [ ] Spacing: Responsive padding/margins (smaller on mobile, larger on desktop)
- [ ] Typography: Font sizes scale appropriately (smaller headers on mobile)
- [ ] Sidebar: Hamburger menu fully functional on mobile (<768px)
- [ ] KPI cards: Stack vertically on mobile, 2-column on tablet, 4-column on desktop
- [ ] Charts: Full-width responsive on mobile, side-by-side on desktop
- [ ] Patient table: Horizontal scroll on mobile, full table on desktop
- [ ] No horizontal overflow (all content visible without side-scrolling)
- [ ] Touch interactions: Buttons/links provide visual feedback on tap (active state)
- [ ] No layout shift on scroll (fixed header + sidebar do not cause jank)
- [ ] Tested on real mobile device (iOS/Android) or mobile emulation

---

## Technical Requirements

**Frontend:**
- Audit all dashboard components for TailwindCSS responsive classes
- Verify responsive breakpoints: `sm:` (640px), `md:` (768px), `lg:` (1024px), `xl:` (1280px), `2xl:` (1536px)
- Touch-friendly interactions:
  - Button/link size: At least 44x44px (iOS guideline)
  - Active state: Darker background or border on tap (`:active` pseudo-class)
  - No hover-only interactions (mobile has no hover)
- Spacing:
  - Mobile padding: `p-4` (16px) for screens < 768px
  - Desktop padding: `p-6` or `p-8` for screens ≥ 768px
- Typography:
  - Mobile: Smaller headings (h3/h4 instead of h1/h2)
  - Desktop: Larger headings (h2/h3)
  - Use TailwindCSS size utilities: `text-lg md:text-2xl`
- No horizontal scroll:
  - Wrap wide tables in `overflow-x-auto` container
  - Ensure all content fits within viewport

**Mobile Browser Testing:**
- Test on Chrome Mobile (via DevTools)
- Test on Safari Mobile (iOS simulator or device)
- Test orientations: Portrait and Landscape
- Test on actual mobile device if possible

**Performance:**
- Lighthouse Mobile score ≥85 (not a blocker, but aim for it)
- No layout shifts (CLS) on navigation
- Smooth scrolling (no janky animations)

**Accessibility Verification:**
- Touch targets: 44x44px minimum
- Color contrast: WCAG AA standard (4.5:1 for text)
- Text size: Readable without zoom (≥16px base)
- Keyboard navigation: Tab through all interactive elements

**Integration:**
- Combine all previous stories (DASH-001 through DASH-005) into single dashboard page
- Verify layout cohesion across all components

**Testing:**
- Manual testing on 3+ screen sizes (mobile, tablet, desktop)
- Chrome DevTools mobile emulation for 375px, 768px, 1440px
- Real device testing (iPhone/Android) if available
- Screenshot comparison: mobile vs. desktop layout
