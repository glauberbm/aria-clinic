## Overview

Set up Supabase Auth integration for ArIA Clinic, implement email/password user registration flow, and establish secure session management with JWT tokens.

---

## User Story Statement

**As a** clinic admin
**I want** to register new users with email and password
**So that** I can manage team access to the clinic management system

---

## Acceptance Criteria

- [ ] Supabase Auth enabled in project with email/password provider
- [ ] User registration form with email validation and password requirements
- [ ] Successful registration creates user in auth.users and public.users table
- [ ] Session management with JWT tokens (auto-refresh, 24-hour expiry)
- [ ] Error handling for duplicate emails and weak passwords
- [ ] Email verification workflow (optional for MVP, log requirement)

---

## Technical Requirements

**Frontend:**
- Create `/app/auth/register` page with form (email, password, password confirmation)
- Form validation using React Hook Form + Zod schema
- Use `@supabase/supabase-js` client for auth operations
- Handle loading states and error messages

**Backend:**
- Enable Supabase Auth with email/password provider
- Create `public.users` table with RLS policy: authenticated users can read own record
- Database schema:
  ```sql
  CREATE TABLE public.users (
    id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    email TEXT UNIQUE NOT NULL,
    name TEXT NOT NULL,
    clinic_id UUID NOT NULL,
    active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT now(),
    updated_at TIMESTAMP DEFAULT now()
  );
  ```
- Create function to sync auth.users → public.users on signup (via Postgres trigger)

**Security:**
- No passwords stored in public.users (managed by Supabase Auth)
- HTTPS only for all auth requests
- Rate limit registration endpoint (max 5 attempts per IP per hour)
- SQL injection prevention (use parameterized queries)

**Testing:**
- Unit test: Form validation with valid/invalid inputs
- Integration test: Register user → verify in database
- Security test: SQL injection attempts, rate limiting
