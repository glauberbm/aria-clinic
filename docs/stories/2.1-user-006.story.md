# User Story: USER-006

**Epic:** EPIC-002 — Patient Management & Health Records
**Title:** Patient Registration & Profile Setup
**ID:** USER-006
**Created:** 2026-05-15
**Status:** Draft

---

## Overview

Implement patient registration system allowing patients to create profiles, upload insurance information, and manage personal health data. Integrates with authentication (USER-001) and RBAC (USER-004).

---

## User Story Statement

**As a** new patient
**I want** to register and create my health profile
**So that** I can use the clinic system and manage my health records

---

## Acceptance Criteria

- [ ] Patient registration form (name, email, phone, date of birth, CPF)
- [ ] Email verification before account activation
- [ ] Patient profile page with editable health information
- [ ] Insurance information storage (policy number, provider, coverage)
- [ ] Medical history summary (conditions, allergies, medications)
- [ ] Profile picture/avatar upload
- [ ] Role assignment: Patient role automatically on registration
- [ ] Email notification to clinic admin on new patient registration
- [ ] Patient can only view/edit own profile (privacy enforcement)

---

## Technical Requirements

**Frontend:**
- Create `/auth/patient-register` page
- Patient registration form with validation
- Email verification flow
- `/app/patient/profile` page with editable sections
- Insurance information form
- Medical history input form

**Backend:**
- Endpoint: POST `/api/auth/patient-register` (registration)
- Endpoint: POST `/api/auth/verify-email` (verification)
- Endpoint: GET `/api/patient/profile` (retrieve own profile)
- Endpoint: PUT `/api/patient/profile` (update profile)
- Endpoint: POST `/api/patient/avatar` (upload avatar)
- RLS policy: Patients can only access own profile data

**Database:**
- Create `patient_profiles` table with extended health info
- Create `insurance_info` table
- Create `medical_history` table (conditions, allergies, medications)
- Implement RLS policies for patient data privacy

**Security:**
- Email verification required before profile access
- RLS enforces patient privacy
- No patient can access other patient's data
- Admin audit trail for new registrations

**Testing:**
- Unit tests: Registration validation, email verification flow
- Integration tests: Complete registration → profile access
- Security tests: Patient privacy enforcement, unauthorized access blocked
