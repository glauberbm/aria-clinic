# User Story: DASH-002

**Created:** 2026-05-15
**Status:** Draft
**Ready for:** @po validation

---

## Overview

Build reusable KPI (Key Performance Indicator) card components that display clinic metrics: total revenue, active patients, scheduled appointments, and pending consultations. Cards show current value and period-over-period change (simulated data, Phase 1).

---

## User Story Statement

**As a** clinic manager
**I want** to see KPI cards displaying key metrics at a glance on the dashboard
**So that** I can quickly assess clinic performance and patient volume

---

## Acceptance Criteria

- [ ] KPI card component created (`/components/dashboard/KPICard.tsx`)
- [ ] Dashboard displays 4 KPI cards: Revenue, Patients, Appointments, Consultations
- [ ] Each card shows: metric name, current value, period (e.g., "This Month"), change indicator (↑/↓) with percentage
- [ ] Cards use mock/static data (no API calls)
- [ ] Card styling: color-coded by metric type (green=positive, red=negative, blue=neutral)
- [ ] Cards responsive: 4-column grid on desktop, 2-column on tablet, 1-column on mobile
- [ ] Hover effect: subtle shadow/scale on desktop
- [ ] Icons: Each KPI has a visual icon (TrendingUp, Users, Calendar, etc.)
- [ ] No loading states (static Phase 1)
- [ ] Accessible: Proper heading hierarchy, aria-labels for icons

---

## Technical Requirements

**Frontend:**
- Create `/components/dashboard/KPICard.tsx` (reusable card component)
- Create mock data structure: `{ label, value, unit, change, changePercent, trend, icon, color }`
- Dashboard page: Display 4 cards in responsive grid
- Grid: `grid-cols-1 md:grid-cols-2 lg:grid-cols-4` (TailwindCSS)
- Icons: `lucide-react` (TrendingUp, Users, Clock, CheckCircle)
- Colors: TailwindCSS palette (green-500, red-500, blue-500, gray-500)
- Hover effect: `hover:shadow-lg hover:scale-105` with transition
